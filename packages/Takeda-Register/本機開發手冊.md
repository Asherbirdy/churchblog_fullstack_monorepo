# 本機開發手冊

## 開發前置設定

為了繞過瀏覽器的 CORS 政策，方便本機串接 API，請使用以下指令開啟一個新的 Chrome 視窗進行開發。

**注意：** 此模式會停用網頁安全性功能，僅供開發使用。

```bash
open -n -a /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --args --user-data-dir="/tmp/chrome_dev_test" --disable-web-security
```

### 1. 啟動專案

在專案根目錄下，執行以下指令來啟動本地開發伺服器：

```bash
pnpm start
2. Takeda-Register
1. dev
```

### 2. 設定 Ngrok (LINE OA 開發)

如果您需要透過 LINE OA 進行本機測試，請遵循以下步驟：

1.  **啟動 ngrok**：
    為您本地正在運行的服務（例如 `http://localhost:5173`）建立一個公開的通道。
    ```bash
    ngrok http --url=submucronated-xxxxx { port }
    ```
    *(請根據 `pnpm start` 啟動的實際 Port 號修改)*

2.  **更新 LINE Developer Callback URL**：
    將 ngrok 提供的 `https` 開頭的轉發網址 (Forwarding URL) 複製並貼到 LINE Developer 後台對應的 Channel 設定中的 Callback URL 欄位。
